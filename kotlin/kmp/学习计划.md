# KMP 一个月学习计划

> **目标：1 个月后达到"会用、能用"的程度**
> 具体标准：能独立搭建 KMP 项目，共享网络层 + 数据层 + 业务逻辑，Android 端可以正常运行。

**起点假设**：Kotlin 语法基础、协程/Flow 已了解、有 Android 原生开发经验。  
**策略**：先跑通 Android + 后端/Desktop 共享，绕开 iOS，1 个月内跑通真实项目。

---

## 总体节奏

| 周次 | 主题 | 关键产出 |
|------|------|---------|
| 第 1 周 | 理解架构 + 搭环境 + 跑第一个项目 | 能跑通官方模板项目 |
| 第 2 周 | 共享网络层 + 数据模型 | 能在 Android/Desktop 共用 API 请求 |
| 第 3 周 | 共享数据库 + ViewModel 逻辑 | 能在多端共用数据持久化和 UI 状态 |
| 第 4 周 | 整合实战 + 查漏补缺 | 完成一个有网络+数据库的完整 Demo |

---

## 第 1 周：理解架构 + 环境搭建

### 目标
- 理解 KMP 是什么、能做什么，和 Flutter/RN 的区别
- 搭建开发环境（Android Studio + KMP 插件）
- 跑通第一个 KMP 项目

### 每日安排

**Day 1（周一）—— 概念理解**
- 阅读官方文档：[KMP 概述](https://kotlinlang.org/docs/multiplatform.html)
- 重点理解：`commonMain / androidMain / iosMain` 源集结构
- 理解 `expect / actual` 机制（KMP 最核心的概念）

**Day 2（周二）—— 环境搭建**
- 安装/更新 Android Studio 到最新版本
- 安装 Kotlin Multiplatform 插件
- 配置 Gradle：`kotlin("multiplatform")` 插件的基本写法
- 可选：安装 Xcode（暂时只跑 Android 也行）

**Day 3（周三）—— 官方模板项目**
- 用 [KMP 向导](https://kmp.jetbrains.com/) 生成模板项目
- 跑通 Android 端
- 读懂项目目录结构：`composeApp/`、`shared/`、`iosApp/`

**Day 4（周四）—— expect/actual 实战**
- 自己写一个 `expect class Platform()`
- 分别在 `androidMain` 和 `desktopMain` 实现 `actual`
- 理解平台相关代码如何被隔离

**Day 5（周五）—— Gradle 配置精读**
- 读懂 `build.gradle.kts` 中的 KMP 配置
- 理解 `sourceSets`、`dependencies` 的写法
- 知道如何添加多平台依赖 vs 单平台依赖

**Day 6~7（周末）—— 整理 + 写笔记**
- 整理本周所学，写到 `1.1_KMP基础概念.md`
- 把 expect/actual 用法举 3 个实际场景例子写下来

---

## 第 2 周：共享网络层

### 目标
- 在 `commonMain` 中写网络请求，Android + Desktop 均可调用
- 掌握 Ktor Client（KMP 官方推荐的多平台 HTTP 库）
- 掌握 Kotlinx Serialization（多平台 JSON 解析）

### 每日安排

**Day 1（周一）—— Ktor Client 入门**
- 添加 Ktor 依赖（注意多平台写法）
- 写第一个 `suspend fun` 网络请求：调用一个公开 API
- 理解 Ktor 的引擎机制（Android 用 OkHttp/Android，Desktop 用 CIO）

**Day 2（周二）—— Kotlinx Serialization**
- 添加 `kotlinx.serialization` 插件和依赖
- 用 `@Serializable` 定义数据模型
- 实现 JSON 解析：`Json.decodeFromString<T>()`

**Day 3（周三）—— 封装 API 层**
- 创建 `ApiService` 接口（在 commonMain 中）
- 封装请求：统一处理 baseUrl、header、error
- 写一个完整的 CRUD 示例

**Day 4（周四）—— Repository 模式**
- 在 `commonMain` 中创建 `Repository` 层
- 依赖注入基础：理解如何在 KMP 中做依赖管理
- 推荐先用简单的手动注入，不引入框架

**Day 5（周五）—— Android 端调用**
- 在 Android `ViewModel` 中调用 `shared` 模块的 Repository
- 用 `viewModelScope` + 协程驱动数据流
- 展示在 Compose UI 中

**Day 6~7（周末）—— 整理 + 写笔记**
- 整理 Ktor 配置写法，写到 `2.1_Ktor网络层.md`
- 踩坑记录：网络层配置中哪些地方容易出错

---

## 第 3 周：共享数据库 + ViewModel 逻辑

### 目标
- 用 SQLDelight 在 KMP 中实现本地数据持久化
- 在 `commonMain` 中写 ViewModel 逻辑（StateFlow 驱动状态）
- 数据流：网络 → Repository → ViewModel → UI

### 每日安排

**Day 1（周一）—— SQLDelight 入门**
- 添加 SQLDelight 依赖（多平台版本）
- 写第一个 `.sq` 文件（SQL 定义文件）
- 理解 SQLDelight 如何自动生成 Kotlin 代码

**Day 2（周二）—— SQLDelight 实战**
- 实现增删改查（CRUD）
- 理解 `DatabaseDriverFactory`（expect/actual 的典型应用）
  - Android：`AndroidSqliteDriver`
  - Desktop：`JdbcSqliteDriver`

**Day 3（周三）—— 共享 ViewModel 逻辑**
- 在 `commonMain` 中写 `ViewModel`（不继承 Android ViewModel）
- 使用 `StateFlow` 管理状态
- Android 端：用 `collectAsState()` 订阅

**Day 4（周四）—— 缓存策略**
- 实现"网络优先，本地缓存兜底"的数据加载逻辑
- 用 Flow 组合网络数据和本地数据

**Day 5（周五）—— 错误处理**
- 统一的 `Result<T>` 封装
- 网络错误 / 数据库错误的统一处理
- 在 UI 层展示 Loading / Success / Error 状态

**Day 6~7（周末）—— 整理 + 写笔记**
- 整理 SQLDelight 配置，写到 `3.1_SQLDelight数据库.md`
- 整理 ViewModel 模式，写到 `3.2_共享ViewModel.md`

## 第 4 周：整合实战

### 目标
- 完成一个包含「网络请求 + 本地缓存 + 状态管理」的完整 KMP Demo
- 总结踩坑经验，写成参考文档

### Demo 项目：「GitHub 用户查询 App」

功能点：
- 输入 GitHub 用户名，查询用户信息 + 仓库列表（Ktor 调用 GitHub API）
- 本地收藏功能（SQLDelight 存储）
- 加载状态展示（StateFlow + Compose）
- Android 可以跑起来

### 每日安排

**Day 1（周一）—— 项目框架搭建**
- 创建新 KMP 项目（不用模板，手动配置）
- 规划模块划分：`data/`、`domain/`、`presentation/`
- 添加所有依赖

**Day 2（周二）—— 数据层实现**
- 实现 GitHub API 接口
- 定义数据模型（User、Repository）
- 实现本地数据库 Schema

**Day 3（周三）—— 业务逻辑层**
- Repository：网络 + 缓存组合
- ViewModel：状态管理
- 错误处理

**Day 4（周四）—— Android UI**
- Compose UI：搜索框 + 用户卡片 + 仓库列表
- 收藏/取消收藏功能
- 完整联调

**Day 5（周五）—— 回顾与总结**
- 查漏补缺：运行中还有哪些报错没处理
- 写项目 README

**Day 6~7（周末）—— 复盘文档**
- 整理整体学习心得，写到 `4.1_KMP实战复盘.md`
- 更新本计划，标记哪些完成、哪些跳过、哪些还需深入
