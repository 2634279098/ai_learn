# ViewModel 到底是什么？（一句话）

ViewModel 就是一个**专门用来存页面数据、处理业务逻辑的类**，**屏幕旋转、键盘弹出、语言切换**这些导致 Activity 重建的情况，它**不会被销毁，数据不会丢**。

---

# 核心作用（面试必背 2 点）

1. **页面旋转 / 配置变更时，数据不丢失**
	- Activity 销毁重建 → ViewModel 还在
	- 数据不用重新加载、不用重新请求
1. **把数据和逻辑从 Activity 里抽出来**
	- Activity 只负责 UI
	- 网络请求、计算、数据处理 → 全放 ViewModel
	- 代码更清晰、不容易崩溃、好维护

---

# 生命周期（超级关键）

- Activity **正常关闭** → ViewModel 销毁
- Activity **旋转 / 配置变更** → ViewModel **不销毁**
- 所以它能**保存数据**

# 最简单代码示例（你能看懂）
```javascript
// 1. 建一个 ViewModel
class MyViewModel : ViewModel() {
    // 存数据
    var count = 0

    // 处理逻辑
    fun add() {
        count++
    }
}
```

```typescript
// 2. 在 Activity 里用
class MainActivity : AppCompatActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)

        // 获取 ViewModel（旋转不会重建）
        val vm = ViewModelProvider(this)[MyViewModel::class.java]

        // 数据一直存在，旋转也不丢
        vm.add()
    }
}
```

# 面试必问 3 题（答案直接背）

### ViewModel 为什么能做到旋转不丢数据？

因为它**独立于 Activity 生命周期**，配置变更时 Activity 重建，但 ViewModel 实例被系统保留。

### ViewModel 能不能存 Context / Activity？

**绝对不能！会内存泄漏！**要用 Context 就用 **AndroidViewModel**（拿到的是 Application）。

### ViewModel 和 onSaveInstanceState 区别？

- **onSaveInstanceState**：存少量简单数据，序列化
- **ViewModel**：存大量 / 复杂数据，不序列化，效率高

---

# 你现在只要记住 3 句

1. ViewModel = **页面数据管家**
1. **配置变更不销毁，数据不丢失**
1. **不持有 Context，避免内存泄漏**
