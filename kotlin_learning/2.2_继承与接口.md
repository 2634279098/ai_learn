## 1. 继承 —— open 关键字、Kotlin 类默认 final

```kotlin
// Kotlin 类默认是 final 的，必须加 open 才能被继承
open class Animal(
    open val name: String,  // open 属性才能被子类重写
    val age: Int
) {
    // open 方法才能被重写
    open fun eat() {
        println("$name 在吃东西")
    }

    // 没有 open 的方法不能被重写
    fun sleep() {
        println("$name 在睡觉")
    }

    // open 属性
    open val description: String
        get() = "动物: $name"
}

// 子类：用 : 表示继承，调用父类构造函数
class Dog(override val name: String, age: Int) : Animal(name, age) {

    // override 重写父类方法
    override fun eat() {
        // super 调用父类的实现
        super.eat()
        println("$name 吃的是骨头")
    }

    // Dog 自己的方法
    fun bark() {
        println("$name: 汪汪汪!")
    }

    // 重写属性
    override val description: String
        get() = "狗狗: $name, $age 岁"
}

class Cat(override val name: String, age: Int) : Animal(name, age) {

    override fun eat() {
        println("$name 在吃鱼")
    }

    // 注意：sleep() 没有 open，不能重写
    // override fun sleep() { }  // 编译错误！
}

// 使用
val dog = Dog("旺财", 3)
dog.eat()       // 调用重写后的方法
dog.bark()      // Dog 自己的方法
dog.sleep()     // 继承自父类，没有重写
println("名字: ${dog.name}, 年龄: ${dog.age}, 描述: ${dog.description}")

val cat = Cat("咪咪", 2)
cat.eat()
cat.sleep()     // Cat 重写了 sleep
```

## 5. 抽象类 —— abstract class

```kotlin
// 5. 抽象类：不能直接实例化，子类必须实现抽象方法
abstract class Shape {
    // 抽象方法：没有方法体，子类必须实现
    abstract fun area(): Double
    abstract fun perimeter(): Double

    // 普通方法：有默认实现，子类可以直接用
    fun printInfo() {
        println("面积: ${area()}, 周长: ${perimeter()}")
    }
}

class Circle(private val radius: Double) : Shape() {
    override fun area() = Math.PI * radius * radius
    override fun perimeter() = 2 * Math.PI * radius
}

class Rectangle(private val width: Double, private val height: Double) : Shape() {
    override fun area() = width * height
    override fun perimeter() = 2 * (width + height)
}

// 使用
val circle = Circle(5.0)
val rect = Rectangle(4.0, 6.0)

println("圆: 面积=${circle.area()}, 周长=${circle.perimeter()}")
println("矩形: 面积=${rect.area()}, 周长=${rect.perimeter()}")
circle.printInfo()  // 抽象类中的普通方法
```

## 6. 接口 —— 可以有默认实现和属性

```kotlin
// 6. 接口：用 interface 定义，可以有默认实现
interface Clickable {
    // 抽象属性（实现类必须提供）
    val isEnabled: Boolean

    // 抽象方法（实现类必须实现）
    fun click()

    // 有默认实现的方法（实现类可以不重写）
    fun longPress() {
        println("默认长按行为：显示提示")
    }
}

// 实现接口：也是用 :（和继承一样的语法）
class Button(private val text: String) : Clickable {
    override val isEnabled: Boolean = true

    override fun click() {
        println("按钮 [$text] 被点击了")
    }

    // longPress 不重写就用接口的默认实现
}

// 使用
val button = Button("提交")
button.click()
button.longPress()
println("是否可用: ${button.isEnabled}")
```

## 7. 接口 vs 抽象类的选择

```
接口：
- 不能有构造函数
- 不能保存状态（属性没有幕后字段，除非自定义 getter）
- 一个类可以实现多个接口
- 适合定义"能力"（Clickable、Serializable、Comparable）

抽象类：
- 可以有构造函数
- 可以保存状态（有幕后字段的属性）
- 一个类只能继承一个抽象类
- 适合定义"是什么"（Animal、Shape、View）
```

## 8. 多接口实现与冲突解决

```kotlin
// 8. 多接口实现与冲突解决
interface Phone {
    fun call() {
        println("打电话")
    }

    fun charge() {
        println("Phone: 用 USB 充电")
    }
}

interface Computer {
    fun browse() {
        println("浏览网页")
    }

    fun charge() {
        println("Computer: 用电源适配器充电")
    }
}

// 同时实现两个接口，charge() 方法冲突了
class SmartPhone : Phone, Computer {
    // 两个接口都有 charge()，必须手动解决冲突
    override fun charge() {
        // 用 super<接口名> 指定调用哪个
        super<Phone>.charge()
        super<Computer>.charge()
        println("SmartPhone: 都支持!")
    }

    // call 和 browse 没有冲突，可以不重写（用默认实现）
    // 也可以选择重写
}

// 使用
val smartPhone = SmartPhone()
smartPhone.charge()  // 冲突方法，看看怎么解决的
smartPhone.call()
smartPhone.browse()
```

## 9. 类型检查与转换 —— is / as / as?

```kotlin
// 9. 类型检查与转换
val animals: List<Animal> = listOf(
    Dog("小黑", 2),
    Cat("小白", 1),
    Dog("大黄", 5)
)

for (animal in animals) {
    // is 检查类型
    when (animal) {
        is Dog -> {
            // 智能转换：编译器知道这里 animal 是 Dog
            animal.bark()
        }
        is Cat -> {
            println("${animal.name} 在睡觉")
        }
    }
}

// as 强制转换（不安全，类型不对会崩）
val animal: Animal = Dog("阿黄", 4)
val myDog = animal as Dog
myDog.bark()

// as? 安全转换（类型不对返回 null）
val maybeCat: Cat? = animal as? Cat
println("安全转换为 Cat: $maybeCat")  // null
```
