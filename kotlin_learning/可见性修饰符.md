# Kotlin 可见性修饰符

> 📖 来源：飞书文档 / Kotlin 学习路线

---

## 1. public —— 默认修饰符，到处可见

```kotlin
// public 是 Kotlin 的默认修饰符（不写就是 public）

// 综合示例类
open class UserProfile(
    val name: String,                    // public（默认）
    val age: Int,                        // public
    protected open val email: String     // protected：子类可访问，外部不行
) {
    // private 方法：只有类内部能调用
    private fun validateEmail(): Boolean {
        return email.contains("@")
    }

    // public 方法：间接暴露 private 逻辑
    fun printEmail() {
        if (validateEmail()) {
            println("邮箱: $email")
        } else {
            println("邮箱格式不对")
        }
    }
}

// 使用
val user = UserProfile("Alice", 25, "alice@example.com")
println(user.name)   // public，随便访问
println(user.age)    // public
// user.email       // 编译错误！protected 属性，类外不能访问
```

## 2. private —— 类内私有 / 文件内私有

```kotlin
// 类级 private：只有类内部能访问
val user = UserProfile("Alice", 25, "alice@example.com")

// user.validateEmail()  // 编译错误！private 方法
user.printEmail()   // OK，通过 public 方法间接访问 private 成员

// 文件级 private：只在当前文件内可见，其他文件不能调用
private fun filePrivateHelper() {
    println("我是文件级 private 函数，只有本文件能调用我")
}

// 文件级 private 类：只在当前文件内可见
private class FilePrivateClass {
    fun hello() = "我是文件级 private 类"
}

// 使用
filePrivateHelper()
```

## 3. protected —— 子类可见（不能用于顶层声明）

```kotlin
// 3. protected：子类可以访问父类的 protected 成员
class Admin(
    name: String,
    age: Int,
    email: String,
    private val role: String
) : UserProfile(name, age, email) {

    fun showRole() {
        // 可以访问父类的 protected 属性 email
        println("管理员: $name, 邮箱: $email, 角色: $role")
    }
}

// 使用
val admin = Admin("Bob", 30, "bob@example.com", "超级管理员")
admin.showRole()
// admin.email      // 编译错误！protected 只有子类能访问，外部不行
```

## 4. internal —— 同模块可见（Kotlin 独有，Java 没有）

```kotlin
// 4. internal：同模块可见
// 如果这个类在一个 library 模块中，app 模块就看不到 internal 成员

class AppSettings {
    internal var cacheSize: Int = 100    // 同模块可见

    internal fun clearCache() {          // 同模块可见
        println("缓存已清除")
    }

    private var secretKey = "abc123"     // 只有类内部可见
}

// 使用
val config = AppSettings()
println("缓存大小: ${config.cacheSize}")  // internal，同模块 OK
config.clearCache()                       // internal 方法，同模块 OK
// 如果这个类在另一个模块（比如一个 library），就访问不到 internal 成员
```

## 5. 与 Java 可见性的对比

| Kotlin      | Java              | 说明                          |
|-------------|-------------------|-------------------------------|
| public      | public            | 一样，但 Kotlin 是默认值       |
| private     | private           | 类似，但 Kotlin 的顶层 private 是文件级 |
| protected   | protected         | Kotlin 不允许同包访问，只有子类 |
| internal    | (无对应)           | Kotlin 独有，同模块可见         |
| (无对应)     | package-private   | Java 默认值，Kotlin 没有这个概念 |

## 6. 构造函数的可见性

```kotlin
// 6. 构造函数设为 private（单例模式的经典写法）
class Singleton private constructor(val name: String) {
    companion object {
        private val instance = Singleton("唯一实例")
        fun getInstance() = instance
    }
}

// 使用
// val s = Singleton()  // 编译错误！构造函数是 private
val s = Singleton.getInstance()
println("单例: ${s.name}")
```

## 7. setter 单独设置可见性

```kotlin
// 7. setter 单独设置可见性
class Counter {
    var count: Int = 0
        private set  // getter 是 public，setter 是 private

    fun increment() {
        count++
    }
}

// 使用
val counter = Counter()
println("初始值: ${counter.count}")  // 读取 OK
// counter.count = 100              // 编译错误！setter 是 private 的
counter.increment()
counter.increment()
println("递增后: ${counter.count}")
```
