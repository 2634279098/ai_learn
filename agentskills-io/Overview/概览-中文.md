# Agent Skills 概览

> 一种为 AI 代理赋予新能力和专业知识的标准化方式。

---

## 什么是 Agent Skills？

Agent Skills 是一种**轻量级开放格式**，用于通过专业知识和工作流扩展 AI 代理的能力。

核心上，一个技能（Skill）是一个包含 `SKILL.md` 文件的文件夹。该文件包含元数据（至少包括 `name` 和 `description`）以及告诉代理如何执行特定任务的指令。技能还可以捆绑脚本、参考材料、模板和其他资源。

```
my-skill/
├── SKILL.md          # 必需：元数据 + 指令
├── scripts/          # 可选：可执行代码
├── references/       # 可选：文档
├── assets/           # 可选：模板、资源
└── ...               # 任何附加文件或目录
```

---

## 为什么需要 Agent Skills？

代理越来越强大，但往往缺乏可靠地完成实际工作所需的上下文。技能通过以下方式解决这一问题——将程序性知识以及公司、团队和用户特定的上下文打包成**可移植的、版本控制的文件夹**，供代理按需加载。这赋予代理：

- **领域专业知识**：将专业知识——从法律审查流程到数据分析管道到演示格式化——捕获为可重用的指令和资源。
- **可重复的工作流**：将多步骤任务转化为一致、可审计的流程。
- **跨产品复用**：构建一次技能，即可在任何兼容技能的代理中使用。

---

## Agent Skills 如何工作？

代理通过**渐进式披露（Progressive Disclosure）**加载技能，分为三个阶段：

1. **发现**：启动时，代理仅加载每个可用技能的名称和描述，仅足以了解何时可能相关。
2. **激活**：当任务与技能描述匹配时，代理将完整的 `SKILL.md` 指令读入上下文。
3. **执行**：代理遵循指令，可选择执行捆绑的代码或按需加载引用的文件。

完整指令仅在任务需要时加载，因此代理可以保留许多技能，而只占用很小的上下文开销。

## 开放开发

Agent Skills 格式最初由 **Anthropic** 开发，作为开放标准发布，已被越来越多的代理产品采用。该标准对更广泛生态系统的贡献开放。
