# Cursor Composer 2.5：Kimi K2.5 底座 + 文本反馈强化学习

## 一句话总结
Cursor Composer 2.5 上线，底座是月之暗面开源的 Kimi K2.5，CursorBench 上超过 Claude Opus 4.7，定价仅为其十分之一，核心训练创新是文本反馈强化学习（在失败的工具调用处给局部提示）。

## 适用场景
- 使用 Cursor 进行日常开发的程序员
- 评估 AI 编程工具选型的团队
- 关注 AI 编程模型训练方法的工程师

## 详细内容

### 模型规格

| 项目 | 数据 |
|------|------|
| 底座模型 | Kimi K2.5（月之暗面开源） |
| 架构 | 万亿参数 MoE，每次推理激活约 320 亿参数 |
| 定价 | 0.50 美元/百万 token 输入、2.50 美元输出 |
| 价格对比 | Opus 4.7 的十分之一 |

### 性能基准

| 基准测试 | Composer 2.5 | Opus 4.7 | GPT-5.5 |
|----------|--------------|----------|---------|
| SWE-Bench Multilingual | 79.8% | 80.5% | — |
| CursorBench v3.1 | 63.2% | 61.6% | — |
| Terminal-Bench 2.0 | 69.3% | 69.4% | 82.3% |

### 核心训练创新：文本反馈强化学习

Cursor 投入 85% 的总算力用于后训练，核心改进包括：

1. **文本反馈强化学习**：在失败的工具调用处给局部提示（而非只在最终结果给奖励），让模型更快从错误中学习。
2. **25 倍于前代的合成训练数据**：大幅提升训练数据的多样性和质量。

### 透明度改进

今年 3 月 Composer 2 上线时，一位开发者在 API 响应中发现了 `kimi-k2p5-rl-0317` 的模型标识符，引发社区对信息透明度的批评。这次 Cursor 在公告首段就写明了底座来源——Kimi K2.5。

### 对开发者的实际影响

- **性价比极高**：Composer 2.5 在 CursorBench 上超过 Opus 4.7，定价仅为其十分之一。如果你是 Cursor 用户，升级到 Composer 2.5 可以显著降低 API 成本。
- **模型选择透明化**：Cursor 首次公开底座来源，这是 AI 编程工具透明度的重要进步。其他工具厂商大概率跟进。
- **关注 Cursor 自研模型**：Cursor 同时宣布正在用 xAI Colossus 2 的百万 H100 等效算力从头训练一个更大的模型，未给出时间表。

### 行业意义

Composer 2.5 证明了开源权重模型（Kimi K2.5）可以通过精细的后训练（特别是文本反馈强化学习），在特定编程任务上超越闭源旗舰模型（Opus 4.7）。这对整个 AI 编程工具行业有深远影响——模型选择不再是「闭源 vs 开源」的简单二元选择，而是「后训练方法 + 底座模型 + 工具链」的综合竞争。
