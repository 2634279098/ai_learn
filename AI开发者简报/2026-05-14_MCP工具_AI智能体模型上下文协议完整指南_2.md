## 一、MCP 核心概念

**模型上下文协议（Model Context Protocol，MCP）** 被誉为"AI 界的 USB-C 接口"，是一种标准化的客户端-服务器架构协议，旨在解决 AI 智能体与外部工具/数据源集成时的碎片化问题。灵感来源于语言服务器协议（LSP）。

### 架构组成

| 组件 | 说明 |
|------|------|
| **MCP 主机** | 您的应用程序或智能体 |
| **MCP 客户端** | 标准化的通信中间层 |
| **MCP 服务器** | 提供具体工具/数据的服务端 |

> **核心优势**：解耦设计，开发者可随意更换模型或工具，无需重写集成逻辑。

---

## 二、MCP 架构的三大支柱

| 类型 | 说明 | 示例 |
|------|------|------|
| **工具** | 可执行的函数，AI 根据用户意图决定调用 | `search_web`、`write_file`、`execute_sql` |
| **资源** | AI 可读取的数据实体（只读） | 文件、API 响应、数据库快照 |
| **提示词模板** | 标准化模板，指导 AI 与特定工具/数据交互 | 确保不同会话中行为一致性 |

## 三、快速入门：FastMCP 构建自定义服务器

### 安装与基础用法

```python
# 安装: pip install fastmcp
from fastmcp import FastMCP

mcp = FastMCP("企业知识库服务器")

@mcp.tool()
def query_inventory(item_id: str) -> str:
    """查询内部数据库中特定商品的库存水平。"""
    return f"商品 {item_id} 当前库存为 42 件。"

@mcp.resource("docs://internal/handbook")
def get_handbook() -> str:
    """返回公司员工手册内容。"""
    return "欢迎加入公司。原则 1：负责任地使用 AI。"

if __name__ == "__main__":
    mcp.run()
```

### 关键要点

- `@mcp.tool()` 装饰器定义可执行工具
- `@mcp.resource()` 装饰器定义只读数据资源（需指定 URI 格式）
- **文档字符串至关重要** —— LLM 依赖它理解工具用途

## 四、与 LangGraph 高阶集成

```python
from langchain_mcp_adapters.tools import load_mcp_tools
from langgraph.prebuilt import create_react_agent
from mcp import ClientSession, StdioServerParameters
from mcp.client.stdio import stdio_client

server_params = StdioServerParameters(command="python", args=["my_server.py"])

async def run_agent():
    async with stdio_client(server_params) as (read, write):
        async with ClientSession(read, write) as session:
            await session.initialize()
            # 自动将 MCP 工具转换为 LangChain 工具格式
            mcp_tools = await load_mcp_tools(session)
            agent = create_react_agent(model, mcp_tools)
            response = await agent.ainvoke({
                "messages": [{"role": "user", "content": "查询 Item-505 的库存状态"}]
            })
```

## 六、MCP vs A2A 协议对比

| 维度 | MCP | A2A（Google） |
|------|-----|---------------|
| **关系类型** | 智能体 ↔ 工具/数据源 | 智能体 ↔ 智能体 |
| **侧重点** | 技术执行与数据获取 | 任务协商、权限委派、多智能体协作 |
| **比喻** | 智能体触碰世界的"双手" | 智能体协作的"社交技巧" |

> **实践建议**：成熟企业架构中应同时使用两者，MCP 提供工具能力，A2A 提供协作能力。

## 七、2026 年开发专家建议

| # | 建议 | 说明 |
|---|------|------|
| 1 | **🔒 安全至上** | MCP 服务器具有文件系统/网络访问权限时，务必在沙箱环境（Docker/gVisor）中运行 |
| 2 | **📝 模式质量** | 文档字符串和 JSON Schema 的质量直接决定 LLM 工具调用效果——描述越详尽越好 |
| 3 | **🚀 混合传输模式** | 本地开发用 `stdio`；云端跨集群部署用 `SSE`（Server-Sent Events） |
| 4 | **⚡ Token 效率优化** | 大型数据集优先使用"资源"而非塞入提示词，让智能体按需通过 MCP URI 获取特定数据 |
