# Cursor 项目开发最佳实践

## 一句话总结
Cursor 不仅是 AI 写代码的工具，更是需要配合软件工程最佳实践的智能协作伙伴，从项目规划到团队协作都有系统方法论支撑。

## 适用场景
- 需要管理多人协作的大型项目
- 希望建立标准化开发流程的团队
- 想深度挖掘 AI 编程工具潜力的开发者
- 需要在项目中复用最佳实践和代码模板

## 详细内容

### 一、项目规划方法

#### 1. 开发前的全面规划
- **文档生成**：使用 Cursor 协助生成项目文档，包括功能规划、数据模型和 API 接口定义
- **团队共享**：将文档保存为 `.md` 文件，放在项目的 `/docs` 目录下，方便团队随时查阅和更新
- **核心理念**：投入时间进行充分的需求梳理和架构设计，通常能显著减少后期的返工时间，避免项目偏离方向

#### 2. 建立代码基础
- 从已有基础开始构建（使用脚手架工具生成初始代码框架），再通过 Cursor 进行优化和完善
- **方法论**："先搭框架，后填细节"——有助于提高代码质量、降低维护成本，让团队更专注于实现业务价值

#### 3. 项目模板与文件结构
- 在标准化的模板和文件结构下能大幅提高开发效率
- 推荐参考 `@CodeGuidedev` 推荐的文件结构，创建一系列清晰的指引文档
- **重要提醒**：如果是现有项目，打开项目之后记得进行代码库的 Index

---

### 二、核心功能使用技巧

#### 4. 定制 AI 行为规则（Rules）
两种配置方式：
- **全局规则**：适用于个人编码习惯，在 `设置 > General > Rules for AI` 中配置
- **项目规则**：在项目根目录创建 `.cursor/rules` 目录，确保团队一致性

#### 5. 上下文模板管理（Notepads）
被许多开发者忽视的强大功能，用于提高 AI 理解精度：
- **代码模式**：存储团队常用的代码模式（登录流程、API 请求模板、错误处理逻辑）
- **规范文档**：保存项目规范文档，随时引用以确保一致性
- **解决方案**：创建问题解决方案库，避免团队成员反复解决相同问题
- **实际效果**：在 Chat 中引用这些模板，能让 Cursor 更精准地理解需求，生成的代码更符合期望

#### 6. 高效编辑快捷方式
| 快捷键 | 功能 |
|--------|------|
| `⌘K` + 选中代码 | 针对性修改现有功能 |
| `⌘K`（不选中） | 快速生成全新代码 |
| `⌘I` | 唤起 AI 聊天框，完成复杂的功能需求 |

#### 7. 版本控制集成
- 在 Cursor 的 Chat 中使用 `@Git` 将 git 提交、差异或拉取请求添加到提示中
- 实现 AI 辅助的版本管理

### 三、团队协作与智能协作技巧

#### 8. 文档与知识管理
- 通过 `设置 > Features > Docs` 添加团队自定义文档
- 让开发过程更加连贯，减少在各种工具间切换的时间损耗

#### 9. 外部工具集成（MCP）
MCP（模型上下文协议）让 Cursor 与外部工具无缝协作：
- **数据库操作**：直接在 IDE 中操作数据库，查询和修改数据
- **AI 分析**：连接 Perplexity AI 等工具获取深入分析

#### 10. 增强型搜索
- **`@web`**：搜索获取最新网络信息，适合查询新技术和解决方案
- **`@Codebase`**：查询探索项目代码库，了解功能实现和依赖关系（复杂项目中理解历史遗留功能时特别有价值）

#### 11. 自动化工作流（Agent 模式）
- **自动搜索和理解相关文件**：利用 AI 快速定位关键文件并理解其功能
- **执行测试和依赖安装**：自动化运行测试用例和管理项目依赖项
- **结合 YOLO 模式**：将多个任务串联执行，适合创建新组件、添加测试用例或处理数据迁移等标准化流程

#### 12. 视觉输入辅助 UI 开发
- 将设计稿截图提供给 Cursor 或结合 MCP 服务
- 获取布局、样式方面的建议或快速生成代码片段
- ⚠️ **注意**：在企业级项目中，AI 的建议需结合项目已有的设计规范和组件库进行评估

#### 13. 编码测试一体化
- 让 AI 在编写完功能后生成初步测试用例并运行
- **流程**：AI 编写代码和测试 → 执行测试修复错误 → 人工审查质量保证
- ⚠️ **注意**：AI 生成的测试往往需要开发者进行细致的评审和补充

### 四、成功的 5 个关键要素

| 要素 | 说明 |
|------|------|
| 🗺️ **重视前期规划** | 投入时间进行架构设计和需求分析 |
| 🛠️ **熟练工具使用** | 掌握 Cursor 的各种功能和快捷方式 |
| 📚 **建立知识体系** | 系统化管理项目文档和开发经验 |
| 🤖 **自动化重复工作** | 让 AI 处理标准化任务，开发者专注创造性工作 |
| ✅ **保持质量意识** | 将测试和优化融入日常开发流程 |
