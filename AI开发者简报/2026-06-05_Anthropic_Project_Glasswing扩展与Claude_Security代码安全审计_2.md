# Anthropic Project Glasswing 扩展：Claude Security 自动化代码安全审计

## 一句话总结
Anthropic 将内部 AI 漏洞扫描能力产品化为 Claude Security（基于 Opus 4.8），已帮助合作伙伴发现 10,000+ 高危/严重漏洞，并计划向安全团队开放漏洞发现工具和开源项目补丁流程。

## 适用场景
- 开发团队希望在日常开发流程中集成 AI 自动化安全审计，提前发现代码漏洞
- 安全工程师需要可扩展的代码库漏洞扫描工具，减少人工审查工作量
- 开源项目维护者关注 AI 辅助的安全漏洞报告和补丁生成流程
- 企业安全团队评估 AI 安全产品的选型

## 详细内容

### 1. Project Glasswing 是什么

Project Glasswing 是 Anthropic 的核心安全协作计划，为关键软件基础设施的维护组织提供早期访问权限，使用其 AI 漏洞扫描模型 **Claude Mythos Preview**。

**规模扩展**：
- 首批（2026 年 4 月）：约 50 个合作伙伴
- 本次扩展（6 月 2 日）：新增约 150 个组织，覆盖 15+ 国家
- 行业涵盖：电力、水务、医疗、通信、硬件等关键基础设施

### 2. Claude Security 产品

与仅限受邀的 Glasswing 不同，**Claude Security 是公开可用的产品**，面向所有开发者和安全团队：

**核心能力**：
- 使用 Claude Opus 4.8 扫描代码库，发现安全漏洞
- 自动生成修复补丁建议，减少人工修复工作量
- 可集成到现有开发工作流中

**实践价值**：
```bash
# 概念性工作流：将 Claude Security 嵌入 CI/CD
# 1. 代码提交后自动触发安全扫描
# 2. Claude Security 分析代码变更，识别漏洞
# 3. 生成修复建议，以 PR Comment 形式反馈
# 4. 开发者审查后合并修复
```

### 3. 验证数据：10,000+ 高危/严重漏洞

截至目前，首批 Glasswing 合作伙伴使用 Mythos Preview 已发现：
- **10,000+ 个高危或严重级别安全漏洞**
- 这些漏洞存在于大型复杂代码库中，此前未被发现
- 证明 AI 模型具备大规模代码库漏洞发现能力

**关键启示**：即使是有专业安全团队的大型组织，其代码库中仍存在大量未被发现的严重漏洞。AI 自动化扫描可以成为"安全网"的最后一层。

### 4. 开源安全与漏洞修复计划

Anthropic 正在解决 AI 漏洞发现后的"最后一步"——验证、披露和修复：

| 计划方向 | 具体行动 |
|----------|----------|
| 开源审核 | 与第三方合作伙伴讨论规模化审核和修补开源软件漏洞 |
| 最佳实践 | 制定向开源维护者披露漏洞的标准化流程 |
| 补丁生成 | 合作伙伴已使用 Mythos Preview 编写补丁 |
| 预发布检查 | 在部署前运行 AI 安全扫描，防止漏洞进入生产 |
| 语言迁移 | 用内存安全语言重写遗留代码库 |

### 5. 安全紧迫性

Anthropic 明确警告：**具备强大网络能力的 Mythos 级 AI 模型将在 6-12 个月内从其他供应商广泛可用**，且可能没有同等安全防护措施。

这意味着：
- 攻击者也将获得类似的 AI 漏洞发现能力
- 防御方必须在此之前部署 AI 驱动的安全工具
- "用 AI 防御 AI"不再是口号，而是时间窗口内的必选项

### 6. 开发者可立即行动的事项

1. **试用 Claude Security**：对自有代码库运行扫描，评估 AI 安全审计效果
2. **集成到 CI/CD**：在代码合并前增加 AI 安全扫描步骤
3. **关注开源安全**：留意 Anthropic 后续发布的开源漏洞审核工具和流程
4. **建立安全基线**：在 AI 攻击能力普及前，完成当前代码库的安全加固
