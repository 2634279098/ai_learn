# OpenAI Codex 锁定态控制 Mac：自主代理能力的重大飞跃

## 一句话总结
OpenAI Codex 新增在 Mac 锁定状态下控制电脑的能力，用户无需保持电脑打开即可让 AI 代理持续运行，标志着 AI 自主代理从「辅助工具」向「自主工作者」的关键转变。

## 适用场景
- 使用 AI 编程工具进行长时间自动化任务的开发者
- 评估 AI Agent 自主化能力和安全边界的团队
- 构建 AI 自动化工作流的工程师

## 详细内容

### 核心能力

| 特性 | 说明 |
|------|------|
| 锁定态控制 | Mac 锁屏后 Codex 仍可操作电脑 |
| 免看守运行 | 用户无需保持笔记本电脑打开 |
| 自主代理 | AI 可在无人干预下持续执行任务 |

### 为什么重要

1. **代理能力质变**：此前的 AI 编程工具（包括 Claude Code、Cursor 等）都需要用户保持电脑活跃状态。Codex 锁定态控制意味着 AI 代理可以真正「无人值守」运行，这是从「辅助工具」到「自主工作者」的关键转变。

2. **工作流影响**：开发者可以让 Codex 在夜间执行长时间任务（大规模重构、测试运行、代码审查等），第二天早上查看结果。这与你之前提出的「AI 自动 Bug 分析」场景高度匹配。

3. **安全隐忧**：锁定态控制也引发了安全风险讨论——如果 AI 代理可以在用户不关注时操作电脑，如何确保其行为安全可控？这与 Anthropic 强调的「安全优先」理念形成对比。

### 对开发者的实际建议

- **优先评估 Codex 企业版**：如果你的工作涉及长时间自动化任务，Codex 的锁定态控制可能大幅提升效率。
- **设置操作边界**：使用 Codex 时建议配置操作白名单，限制代理在无人值守时可以执行的操作范围。
- **关注 Claude Code 对应能力**：目前 Claude Code 尚未支持类似功能，但 Anthropic 可能在后续版本中加入。

### 行业背景

OpenAI 在 5 月 16 日重组中将 ChatGPT、Codex、API 三线合一，Brockman 统一产品战略，目标是构建「一个统一的 Agent 平台」。锁定态控制是这个统一平台的重要一步。
