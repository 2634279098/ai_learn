# Cursor 高级使用技巧

## 一句话总结
掌握Cursor的10个高级技巧，可显著提升AI辅助编程效率，实现多文件编辑、上下文精准管理、成本有效控制，让开发效率提升40%-60%。

## 适用场景
- 多文件功能重构、跨模块Bug修复
- 大型项目上下文管理，避免AI"失忆"
- 自定义常用操作命令，减少重复劳动
- 控制AI辅助编程成本，优化模型使用策略

## 详细内容
### 一、核心快捷键技巧
1. **Cmd/Ctrl+I：Composer全屏模式**
   - 按`Cmd+I`（Mac）或`Ctrl+I`（Windows）打开，支持`@`引用多个文件，一次性输入多文件编辑需求
   - 示例：`@frontend/RegisterForm.tsx @backend/auth.controller.ts 重构用户注册功能，添加邮箱验证`
   - 效果：多文件编辑效率提升3倍
2. **@符号的8种高级用法**
   - `@文件名`：引用特定文件；`@文件夹`：引用整个目录；`@代码块`：选中代码后@，精准到行
   - `@Git`：引用最近的Git变更；`@Web`：搜索网络资料；`@Codebase`：搜索整个代码库
   - 效果：AI准确率从62%提升到91%
3. **Ctrl/Cmd + →：部分接受AI建议**
   - AI生成代码后，按此快捷键逐段接受建议，不需要的部分按`Esc`跳过
   - 效果：编码速度提升40%，鼠标操作减少28%

### 二、上下文管理技巧
1. **.cursorrules文件配置**
   - 在项目根目录创建`.cursorrules`，定义技术栈、代码规范、禁止事项
   - 示例配置：
     ```
     技术栈：React 18 + TypeScript + Vite
     代码规范：组件使用函数式+Hooks，文件名用PascalCase，禁用var关键字
     禁止使用：class组件、jQuery
     ```
   - 效果：PR评审意见减少70%，TypeScript错误降低35%
2. **长上下文管理策略**
   - 使用Notepads功能记录关键信息和设计决策
   - 定期总结对话并新建会话，避免对话过长导致AI"失忆"
   - 复杂任务分阶段执行，大功能拆成小任务
   - 效果：长期项目中AI准确率提升45%
3. **Plan Mode（计划模式）**
   - 在Composer模式下点击"Plan"按钮，或要求AI"先制定计划"
   - AI会生成详细执行计划（数据库设计→后端API→前端界面→测试）
   - 效果：复杂任务完成率提升60%，返工率降低40%

### 三、Prompt优化技巧
1. **自定义Commands**
   - 在Settings→Commands中添加常用命令，如：
     - `/review`：审查选中代码，检查性能、安全、规范问题
     - `/test`：为选中函数生成单元测试，覆盖率≥80%
     - `/refactor`：重构选中代码，优化时间复杂度和可读性
   - 效果：重复操作效率提升80%
2. **高效Prompt黄金公式**
   ```
   具体任务 + 技术要求 + 上下文引用（@符号）+ 预期结果
   ```
   - 示例：`使用JWT实现登录功能，包含邮箱验证、密码加密（bcrypt）、token刷新机制，参考@auth/login.ts的代码风格`

### 四、成本控制与故障恢复
1. **模型选择策略**
   - GPT-4：复杂架构设计、关键Bug修复、新功能规划
   - Claude Sonnet：日常编码、代码审查、重构优化（性价比最高）
   - GPT-3.5-turbo：简单修改、代码格式化、生成注释
   - 效果：月度费用从28刀降到15刀，节省45%成本
2. **版本控制与故障恢复**
   - 每次重要修改前先提交Git：`git add . && git commit -m "AI重构前备份"`
   - 使用Cursor的Diff View查看AI修改，启用自动备份功能
   - 故障恢复流程：`git diff`查看变更→定位问题文件→`git checkout HEAD -- 文件名`回滚
