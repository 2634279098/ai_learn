# OpenAI Codex 桌面应用全场景升级

## 一句话总结
Codex 从"终端里的 AI 编码助手"升级为覆盖电脑操作、浏览器交互、图像生成、长期记忆、自动化调度的全场景开发代理，将 AI 编程从"写代码"拓展到"操作电脑上的一切"。

## 适用场景
- 需要跨代码、浏览器、本地应用、图像的完整软件交付流程
- 需要测试没有 API 的桌面应用或界面流程
- 需要让 AI 参与前端设计稿、产品概念图等视觉工作
- 长期跨天/跨周的项目跟踪和自动化
- 需要在非本地环境（远程 devbox）中进行开发

## 详细内容

### 核心升级：六大新能力

#### 1. Computer Use（电脑操作）
Codex 现在可以在你的电脑上直接操作应用——看屏幕、点击、输入，并使用自己的光标执行任务。首先在 macOS 上可用。

**最佳实践：**
- 调整前端界面后，直接在真实应用中点选和检查结果
- 测试没有 API 的桌面应用或界面流程
- 不打断当前工作的前提下，让多个 agent 并行执行不同任务

#### 2. 原生浏览器与网页交互
桌面应用内置浏览器，可直接在页面上评论或标注，让代理更精确理解修改意图。

**最佳实践：**
- 前端开发时在 localhost 上快速迭代页面和交互
- 游戏开发中实时预览和调整
- 直接标注 UI 元素让 AI 精准定位对应代码

#### 3. 图像生成能力
集成 `gpt-image-1.5`，可在开发流中直接生成产品概念图、前端稿、游戏素材。关键不只是"能画图"，而是图像、截图和代码可以放在同一个工作流里联动。

**适用场景：**
- 产品概念视觉
- 前端设计稿和 mockup
- 游戏素材生成

#### 4. 深度开发工作流支持
- **GitHub Review Comments**：直接处理 GitHub review comments，将"写代码"和"根据审查意见修正代码"连成闭环
- **多终端标签页**：同时运行多个 terminal tabs，并行查看日志、测试、构建输出
- **SSH 连接远程 Devbox**：直接操作远程 Linux 开发机、云开发盒子（alpha 阶段）
- **富预览侧边栏**：直接打开 PDF、电子表格、Slides、文档，将 PRD、设计说明等纳入 agent 工作上下文
- **Summary Pane**：跟踪 agent 的计划、来源和产出，增强长链路工作流的可追踪性

#### 5. 长期任务与自动化
- **复用已有线程**：automations 可复用现有对话线程，避免从零开始建立上下文
- **安排未来任务**：Codex 可为自己安排未来工作，时间跨度可达数天乃至数周
- **典型用途**：定期跟进未合并的 PR、持续追踪任务回复、在多工具对话中保持上下文同步

#### 6. Memory（记忆）与主动建议
- 记住个人偏好、用户给过的纠正、之前收集的信息
- 综合项目上下文、已连接插件和记忆，主动建议"接下来该做什么"

### 90+ 新增插件
连接了更多开发工具，包括：Atlassian Rovo, CircleCI, CodeRabbit, GitLab Issues, Microsoft Suite, Neon by Databricks, Remotion, Render, Superpowers 等。

### 开发者最佳实践
1. **将 Codex 视为工作流伙伴而非单点工具**——不只是生成代码，更要让它参与完整软件交付过程
2. **利用长期记忆与线程复用减少重复工作**——对跨天 PR 或长期工单复用已有会话线程
3. **在非 API 场景下使用 Computer Use**——测试没有 API 的桌面应用
4. **整合非代码文件进入上下文**——利用富预览功能将 PRD、设计说明纳入工作流
5. **利用 Summary Pane 追踪长链路任务**——对跨工具、多步骤复杂任务确保透明可控
